# Загрузчик вложений из почты (АРХИМЕД → сетевая папка)

Скрипт автоматически подключается к почтовому ящику `muzgb8@sochi.com` (Яндекс), ищет **непрочитанные письма** от отправителя `result@arhimed.clinic`, сохраняет вложения (PDF, Excel, Word) в сетевую папку `\\10.20.20.2\Папка обмена\Cytology` и помечает письма как прочитанные.

## Где работает

- **Сервер:** LXC-контейнер на PVE `x3250m6` (IP: 10.6.1.5), внутри Debian 12.
- **Сетевая папка** смонтирована на хосте PVE в `/mnt/share_host` и проброшена в контейнер через bind mount в `/mnt/share`.
- **Планировщик:** cron, запуск каждые 5 минут. Логи пишутся в `/var/log/mail_downloader.log`.

## Быстрая проверка состояния

```bash
# Посмотреть последние запуски
tail -f /var/log/mail_downloader.log

# Проверить, что cron работает
systemctl status cron

# Посмотреть сохранённые файлы
ls -la /mnt/share/Cytology/
```

## Как остановить / запустить вручную

# Отключить авто-запуск (закомментировать строку в crontab)
crontab -e   # поставить # в начале строки */5 * * * * ...

# Запустить разово
python3 /opt/mail_downloader/mail_downloader.py
```

## Настройка нового сервера (если потребуется)

1. Установить пакеты: `apt install python3 cifs-utils`
2. Скопировать проект из Git: `git clone https://github.com/lyamancini-dev/mail-attachment-downloader.git /opt/mail_downloader`
3. Создать `config.py` на основе `config.example.py`, указав реальные пароли.
4. Смонтировать сетевую шару на хосте PVE и настроить bind mount в контейнер (см. внутреннюю документацию).
5. Добавить задачу в crontab:  
   `*/5 * * * * /usr/bin/python3 /opt/mail_downloader/mail_downloader.py >> /var/log/mail_downloader.log 2>&1`

## Важные детали для администрирования

- **Пароль приложения Яндекса** (не пароль от почты) - менять только в `config.py`.
- **Сетевая учётка** для доступа к папке обмена - `стандартный логин.пароль`. Если пароль изменится - обновить в `config.py`.
- **Папка назначения** - `/mnt/share/Cytology`. Должна быть доступна на запись. Если нет - создать руками: `mkdir -p /mnt/share/Cytology`.
- **Лог-файл** может разрастись - настроить ротацию через `logrotate` при необходимости.
